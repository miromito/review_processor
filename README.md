# Анализ отзывов

Веб-приложение для загрузки отзывов (CSV, JSON или публичная **Google Таблица**), настройки колонок, пакетного анализа через **OpenAI** (тональность, темы, ключевые слова, обоснование) и просмотра результатов: графики (**Chart.js**), таблица с фильтрами, автоматическое бизнес-заключение. Данные хранятся в **MongoDB**.

Требования: **Python 3.12**, **MongoDB 7**, для анализа — ключ **`OPENAI_API_KEY`**.

---

## Возможности

### Проекты и данные

- Несколько изолированных **проектов** с названием, фазой жизненного цикла и метаданными (файл, число строк, даты создания/обновления).
- **Два источника данных**: загрузка **CSV/JSON** с устройства или импорт по ссылке на **публичную Google Таблицу** (CSV export).
- Ограничения загрузки: размер файла (`MAX_UPLOAD_MB`), лимит строк (`MAX_IMPORT_ROWS`), для таблиц — таймаут и максимальный размер ответа (`SHEET_FETCH_TIMEOUT_S`, `SHEET_MAX_BYTES`).
- **Детерминированный хэш строки** (`content_hash`) для учёта содержимого при синхронизации таблицы — новые строки добавляются, дубликаты по содержимому не дублируются.

### Пайплайн проекта (фазы)

1. **Ожидание файла/таблицы** — загрузка или импорт, предпросмотр колонок.
2. **Настройка колонок** — колонка текста отзыва (обязательно), дата (опционально), несколько колонок **фильтров** (мультивыбор).
3. **Готов к анализу** — после сохранения маппинга показывается, сколько строк (**M**) в файле и сколько (**K**) попадает в лимит токенов **`TOKEN_LIMIT_T`** (подсчёт токенов по модели).
4. **Анализ** — фоновая задача: статусы `queued` → `running` → `completed` / `failed`, опрос через API `GET /api/projects/jobs/{job_id}`.
5. **Готово** — дашборд, таблица, инсайт; **ошибка** — сообщение и возможность перезапуска.

### Анализ (OpenAI)

- Пакетная обработка строк с **адаптивным разбиением на батчи** по бюджету токенов на запрос (`ANALYSIS_BATCH_TOKEN_BUDGET`).
- Для каждой строки: **тональность** (positive / negative / neutral), **темы** из фиксированного словаря (размер словаря настраивается **`topic_count`** в диапазоне 3–20), до **четырёх ключевых слов**, **краткое обоснование** (`rationale`).
- Схема ответа модели задаётся через structured output; словарь тем наращивается по мере прохода батчей.
- После успешного полного анализа — генерация и сохранение **краткого бизнес-инсайта** по сводкам и выборке строк (`/api/projects/{id}/insight`).

### Дашборд и графики

- **Сводка тональности** — кольцевая диаграмма (доли позитив / негатив / нейтрал / прочее).
- **Краткий бизнес-инсайт** на карточке рядом с пирогом; длинный текст можно развернуть («Показать полностью»).
- **Динамика тональности** — при наличии колонки даты: столбцы по дням + вторая ось с долей позитивных и скользящим средним.
- **Темы и тональность** — stacked bar по темам.
- **«Боль»** — темы с упором на негатив (объём, доля негатива, индекс).
- **Облако ключевых слов** (до 14 позиций); **щелчок по слову** включает фильтр дашборда по этому ключу (чип с возможностью снять).
- **Пузырьковый график** (scatter): дата × тональность, размер/цвет по теме; группировка по оси времени (**день / неделя / месяц / квартал / год**); клик по дню — список отзывов за период (`/api/projects/{id}/reviews-by-date`).
- **Offcanvas «Фильтры»** для графиков: диапазон дат, тема, значения пользовательских колонок-фильтров из файла (facets подгружаются с API).
- Сворачиваемые блоки **«Подсказка»** (`<details>`) у графиков с пояснением осей и интерпретации.

### Таблица «Отзывы»

- Постраничная выдача с **фильтрами**: тональность, темы (несколько полей), произвольный текст, дата, ключевые слова, значения колонок из файла.
- **Facets** (`GET .../results/facets`) — списки значений для селектов.
- Разворачивание длинного текста отзыва, массовое «развернуть/свернуть».
- Отдельная вкладка **«Аналитика»** с удобным доступом к инсайту и связанным фильтрам.

### Google Таблицы

- Импорт CSV по **export-ссылке** (внутреннее преобразование URL таблицы).
- На завершённом проекте: кнопка **«Проверить таблицу»** / **ручная синхронизация** (`POST .../sync-spreadsheet`) — дозагрузка новых строк и **инкрементальный анализ** только новых отзывов с использованием уже сохранённого **словаря тем**.
- **Фоновый опрос** раз в минуту: для проектов с `data_source: spreadsheet`, фазой `complete` и заданным интервалом **`sync_interval_minutes`** (ограничение снизу/сверху в коде) выполняется автоматическая проверка и при появлении новых строк — постановка инкрементальной задачи.

### Уведомления по e-mail (опционально)

- Настройка **SMTP** (`SMTP_*`, `MAIL_FROM`, `APP_BASE_URL` для ссылок в письмах).
- В маппинге проекта: **e-mail для уведомлений** — письмо при **успешном завершении** или **ошибке** полного анализа.
- Для таблиц: опция **алерта по доле негатива** среди **только что обработанных** новых строк (порог в процентах).

### Список проектов (UI)

- Таблица (десктоп) и **карточки** (мобильная версия), **поиск по названию**.
- Колонка **«Статус»**: спиннер при анализе, иконка успеха при «Готово», предупреждение при промежуточных фазах, ошибка при сбое.
- Во время **анализа** нельзя открыть проект из списка (нет ссылки и кнопки «Открыть»); при **ошибке** кнопка «Открыть» скрыта, переход по **названию** сохраняется для диагностики и повтора.
- **Меню действий** (три точки): переименовать, экспорт CSV, синхронизация таблицы (если источник — spreadsheet), **поделиться** (копирование ссылки в буфер), удалить.
- **Свайп** на мобильных для вызова удаления.
- После редиректа со страницы проекта во время чужого сеанса анализа — **toast** с пояснением (через `sessionStorage`).
- На странице проекта: если открыть URL анализируемого проекта **без** локального флага сессии запуска анализа — **редирект на главную**; при опросе джоба показывается дружелюбный индикатор вместо сырого «Статус: running».

### Экспорт и API

- **`GET /api/projects/{id}/export.csv`** — выгрузка результатов анализа в CSV (с полями по строкам).
- **`PATCH /api/projects/{id}/name`** — переименование.
- REST API для проектов: создание, детали, загрузка, импорт таблицы, синхронизация, маппинг, запуск анализа, агрегаты, дашборд, scatter, отзывы по дате, результаты с пагинацией, инсайт, удаление.
- Интерактивная документация: **`/docs`** (Swagger).

### Безопасность (опционально)

- Если заданы **`AUTH_USERNAME`** и **`JWT_SECRET`**, включается вход по логину/паролю: **JWT в httpOnly cookie**, middleware для HTML и `/api`, страница **`/login`**, выход через **`/api/auth/logout`**.
- Без пары логин+секрет приложение работает **без авторизации** (как в режиме разработки).

### Инфраструктура

- **FastAPI**, шаблоны **Jinja2**, статика и скрипты в `static/`, стили поверх **Bootstrap 5**.
- Индексы MongoDB на старте (`ensure_indexes`).
- Версия приложения в подвале через **`APP_VERSION`** (например, короткий SHA при сборке Docker).

## Запуск без Docker

```bash
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
```

В `.env` минимум задать `MONGODB_URI` и `OPENAI_API_KEY`, затем:

```bash
uvicorn app.main:app --reload --port 8000
```

Сайт: http://127.0.0.1:8000/ — список проектов, создание — **«Новый проект»**.

### Тесты

```bash
pytest
```

## Docker

Скопировать `.env.example` → `.env`, задать учётные данные Mongo и ключ OpenAI, затем:

```bash
docker compose up --build
```

Локально: `docker compose` пробрасывает **127.0.0.1:8000** (приложение не слушает публичный 80). На Droplet снаружи — **nginx + HTTPS**; пошагово: **`deploy/SSL.md`**. Mongo доступна только внутри сети compose.

Если том Mongo уже создавался без авторизации, с новыми `MONGO_INITDB_*` стек может не подняться — тогда пересоздать том (данные будут потеряны).

## Структура репозитория

| Путь | Назначение |
|------|------------|
| `app/main.py` | Точка входа FastAPI, middleware JWT, раздача страниц, lifespan (в т.ч. цикл опроса Google Таблиц) |
| `app/routers/` | Маршруты API (`projects`, `auth`) |
| `app/services/` | Анализ, джобы, дашборд, инсайт, файлы, CSV, синхронизация таблиц, почта, токены, хэш строк |
| `app/templates/` | HTML-шаблоны |
| `static/` | CSS, JS страниц |
| `.github/workflows/` | CI/CD при необходимости |
